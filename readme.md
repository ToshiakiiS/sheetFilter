## 🇬🇧 English Version

A high-performance, adaptive Streamlit application featuring a **hybrid smart parsing engine**. This tool is specifically built to handle normal spreadsheets gracefully while offering an advanced XML data-mining fallback to reconstruct corrupted or massive Excel files generated by legacy ERP systems.

### ✨ Features

* **Adaptive Hybrid Engine:**
    * **Strategy 1 (Native):** Attempts standard `pandas` and `openpyxl` reading for 100% compatibility with healthy files.
    * **Strategy 2 (Emergency XML Mining):** If native reading fails due to structural corruption or broken pointers (common in huge 130MB+ ERP exports), it shifts into a memory-efficient XML stream parser (`xml.etree.ElementTree.iterparse`) to extract raw text directly from the underlying file structure.
* **Smart Sheet Selection:** Automatically detects and targets the largest worksheet by byte size inside compressed `.xlsx` structures.
* **Corrupted Byte Stripping:** Cleans out electronic junk or misaligned headers injected by legacy ERP software by targeting the true ZIP header (`PK\x03\x04`).
* **Interactive UI:** Dynamically displays available columns, allows multi-select filtering, and handles state management to prevent repetitive processing overhead.
* **Safety Guardrails:** Enforces Excel’s hard limit of **1,048,576 rows**, warning the user and advising a CSV export when the threshold is crossed.

### **📖 How it Works Under the Hood**

**The ERP Corruption Problem:** Many enterprise systems export "Excel" files that are actually malformed XML structures zipped together loosely. Standard parsers crash immediately on these files due to unclosed tags or broken index mappings.

This application circumvents that roadblock entirely by treating the `.xlsx` file as the zip archive it actually is:

Plaintext

```
[ Upload File ] 
       │
       ├──► Try Standard Pandas (Fast & Native) ──► Success!
       │
       └──► [On Crash] Fallback to XML Mining
                 │
                 ├── Strip non-ZIP byte junk
                 ├── Extract Shared Strings mapping
                 └── Stream-parse the largest internal worksheet
```

### **⚙️ Configuration & Performance Notes**

* **Memory Management:** The XML parser uses `.clear()` on tree roots during iteration. This keeps memory footprints remarkably low, even when processing files over 100MB directly in the browser.  
* **`st.session_state`:** The app caches the read bytes and generated file outputs so that changing the output format or clicking download multiple times doesn't trigger a full re-parse of the dataset.  
* **`EXCEL_ROW_LIMIT`**: Set to `1,048,576`. If your filtered dataset exceeds this size, the application forces a CSV output format to prevent data truncation.

## **🇧🇷 Versão em Português (PT-BR)**

Uma aplicação Streamlit de alta performance equipada com um **motor híbrido de leitura inteligente**. Este sistema foi projetado para processar planilhas normais com máxima compatibilidade e, ao mesmo tempo, oferecer uma contingência avançada de mineração de XML para reconstruir arquivos corrompidos ou gigantescos extraídos de sistemas ERP legados.

### **✨ Funcionalidades**

* **Motor Híbrido Adaptativo:**  
  * **Estratégia 1 (Nativa):** Tenta ler o arquivo usando as bibliotecas padrão `pandas` e `openpyxl` para garantir 100% de compatibilidade com planilhas saudáveis.  
  * **Estratégia 2 (Mineração de Emergência):** Se o arquivo estiver corrompido (erros de XML estrutural ou ponteiros quebrados, comum em relatórios pesados de mais de 130MB), o motor ativa um parser de fluxo (`xml.etree.ElementTree.iterparse`) que minera os dados textuais diretamente da estrutura bruta do arquivo compactado.  
* **Seleção Automática de Aba:** Varre a estrutura interna do arquivo `.xlsx` e seleciona automaticamente a maior aba em tamanho de bytes, eliminando a necessidade de adivinhação.  
* **Limpeza de Lixo Eletrônico de ERPs:** Remove automaticamente bytes corrompidos ou relatórios de cabeçalho fantasma injetados por sistemas legados, localizando o cabeçalho ZIP real do arquivo (`PK\x03\x04`).  
* **Interface Dinâmica:** Mapeia as colunas do arquivo em tempo de execução, permitindo seleção flexível (inclusão/exclusão) através de um componente multiselect amigável.  
* **Trava de Segurança de Linhas:** Respeita o limite físico do Excel de **1.048.576 linhas**. Caso o arquivo filtrado ultrapasse esse limite, o sistema bloqueia a geração do `.xlsx` e orienta o usuário a exportar em CSV.

### **📖 Como o Motor Funciona por Trás dos Panos**

**O Problema dos ERPs:** É muito comum que sistemas corporativos exportem arquivos com a extensão `.xlsx` que, na verdade, possuem estruturas XML malformadas ou tags mal fechadas. Isso faz com que leitores tradicionais travem ou quebrem imediatamente.

Esta ferramenta resolve o problema tratando o arquivo `.xlsx` como o arquivo compactado (ZIP) que ele realmente é:

Plaintext

```
[ Upload do Arquivo ] 
       │
       ├──► Estratégia 1: Leitura Nativa (Rápida e Direta) ──► Sucesso!
       │
       └──► [Se Falhar/Corrompido] Estratégia 2: Mineração XML
                 │
                 ├── Remove lixo eletrônico antes do header ZIP
                 ├── Extrai o dicionário global de textos (Shared Strings)
                 └── Faz o stream-parse apenas da maior aba interna
```

### **⚙️ Notas de Configuração e Performance**

* **Gerenciamento de Memória:** Durante a mineração da Estratégia 2, o parser limpa os nós da árvore XML da memória (`root.clear()`) logo após processar cada linha. Isso garante um consumo de memória extremamente baixo, permitindo processar arquivos grandes diretamente no navegador.  
* **`st.session_state`:** O aplicativo armazena o cache dos bytes consultados e do arquivo gerado para evitar o reprocessamento da planilha caso o usuário mude apenas o formato de exportação ou clique no botão de download repetidamente.  
* **`EXCEL_ROW_LIMIT`**: Fixado em `1.048.576`. Se a sua base filtrada passar desse tamanho, a aplicação força a saída em CSV para evitar truncamento de dados.
